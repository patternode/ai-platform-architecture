# AI Agent Access to Snowflake Data Products
## Architecture Options Analysis & Recommendations

**Document Version:** 1.0  
**Date:** December 2025  
**Status:** Recommendation for Decision  

---

## Executive Summary

This document presents a comparative analysis of two architectural approaches for enabling AI agents to access Snowflake data products:

- **Option 1:** Power BI Semantic Model with Scheduled Refresh (Current Proposal)
- **Option 2:** Snowflake-Native Semantic Layer with MCP (2025 Best Practice)

Based on the organization's requirements for multi-agent support (Claude, AWS AgentCore, Salesforce Agentforce), real-time data access, majority Snowflake budget allocation, and OSI compliance, **Option 2 is recommended**.

- Data pulled from Snowflake on schedule via Gateway
- Stored in centralized Power BI Semantic Model (Import mode)
- Reports and Copilot query Power BI model, not Snowflake directly
- Reduces Snowflake compute by caching data in Power BI
- Business logic defined in DAX within Power BI

**Limitations:**
- Only Power BI Copilot can access the semantic model
- No support for external AI agents (Claude, AgentCore, Agentforce)
- Data latency based on refresh schedule
- Semantic model not portable or reusable outside Microsoft ecosystem

| User adoption resistance | Medium | Medium | Training, change management, executive sponsorship |

## Appendix A: Semantic View YAML Example

```yaml
# Example Snowflake Semantic View Definition (OSI-Compliant)
name: sales_analytics
description: "Sales performance semantic model for AI agent access"
owner: data_team@company.com

tables:
  - name: orders
    description: "Customer order transactions"
    base_table:
      database: PROD_DB
      schema: SALES
      table: FACT_ORDERS
    
    dimensions:
      - name: order_date
        description: "Date the order was placed"
        expr: ORDER_DATE
        data_type: DATE
        synonyms: 
          - "date"
          - "when ordered"
          - "purchase date"
          - "transaction date"
        
      - name: customer_segment
        description: "Customer classification tier (Enterprise, SMB, Consumer)"
        expr: CUSTOMER_SEGMENT
        data_type: VARCHAR
        synonyms:
          - "segment"
          - "customer type"
          - "tier"
          - "classification"
        sample_values:
          - "Enterprise"
          - "SMB"
          - "Consumer"
        
      - name: product_category
        description: "High-level product grouping"
        expr: PRODUCT_CATEGORY
        data_type: VARCHAR
        synonyms:
          - "category"
          - "product type"
          - "product group"
    
    measures:
      - name: total_revenue
        description: "Total revenue in USD (net of discounts)"
        expr: SUM(ORDER_AMOUNT)
        data_type: NUMBER
        synonyms:
          - "revenue"
          - "sales"
          - "total sales"
          - "income"
          - "turnover"
        
      - name: order_count
        description: "Number of distinct orders"
        expr: COUNT(DISTINCT ORDER_ID)
        data_type: NUMBER
        synonyms:
          - "orders"
          - "number of orders"
          - "order volume"
          - "transaction count"
        
      - name: average_order_value
        description: "Average revenue per order"
        expr: SUM(ORDER_AMOUNT) / NULLIF(COUNT(DISTINCT ORDER_ID), 0)
        data_type: NUMBER
        synonyms:
          - "AOV"
          - "average order"
          - "avg order value"

  - name: customers
    description: "Customer master data"
    base_table:
      database: PROD_DB
      schema: SALES
      table: DIM_CUSTOMER
    
    dimensions:
      - name: customer_name
        description: "Customer account name"
        expr: CUSTOMER_NAME
        data_type: VARCHAR
        synonyms:
          - "name"
          - "account"
          - "client name"
      
      - name: region
        description: "Geographic region"
        expr: REGION
        data_type: VARCHAR
        synonyms:
          - "geography"
          - "territory"
          - "area"

relationships:
  - name: orders_to_customers
    left_table: orders
    right_table: customers
    relationship_type: many_to_one
    join_columns:
      - left_column: CUSTOMER_ID
        right_column: CUSTOMER_ID

verified_queries:
  - question: "What was total revenue last month?"
    sql: |
      SELECT SUM(ORDER_AMOUNT) as total_revenue
      FROM PROD_DB.SALES.FACT_ORDERS
      WHERE ORDER_DATE >= DATEADD(MONTH, -1, DATE_TRUNC('MONTH', CURRENT_DATE()))
        AND ORDER_DATE < DATE_TRUNC('MONTH', CURRENT_DATE())
    
  - question: "Show me revenue by customer segment"
    sql: |
      SELECT CUSTOMER_SEGMENT, SUM(ORDER_AMOUNT) as revenue
      FROM PROD_DB.SALES.FACT_ORDERS
      GROUP BY CUSTOMER_SEGMENT
      ORDER BY revenue DESC
```

## Appendix B: Glossary

|------|------------|
