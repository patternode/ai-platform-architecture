# Visual Design Standards for AI Agents

##  Summary

This document consolidates all visual design standards for AI agents creating artifacts for Bank of New Zealand (BNZ). All AI agents MUST follow these standards when generating:
- Draw.io diagrams
- Any visual artifacts

**Authority**: These standards are mandatory for all AI-generated content and are enforced through automated validation.

### Color Usage Rules

#### ✅ DO:
- Use **BNZ Navy Blue** (`#003087`) for primary headers and key elements
- Use **BNZ Orange** (`#FF6B35`) sparingly for accents and CTAs
- Use **BNZ Teal** (`#00A651`) for success states
- Ensure **4.5:1 contrast ratio** minimum for text (WCAG AA)
- Use semantic colors consistently (green=success, red=error)

#### ❌ DON'T:
- Don't use colors outside the approved palette without justification
- Don't use BNZ Orange as a background color (insufficient contrast)
- Don't use light blue text on white backgrounds (accessibility fail)
- Don't mix random colors - follow the palette hierarchy

## Typography Standards

### Font Families

| Context | Primary Font | Fallback | Use Case |
|---------|-------------|----------|----------|
| **Documents** | Calibri | Arial, sans-serif | Body text, headings |
| **Presentations** | Calibri | Arial, sans-serif | All text content |
| **Diagrams** | Helvetica | Arial, sans-serif | Labels, annotations |
| **Code/Technical** | Consolas | Courier New, monospace | Code blocks, technical specs |

### Font Sizes

#### Documents (Markdown/Word)
- **H1 (Document Title)**: 28-32pt, Bold, BNZ Navy Blue
- **H2 (Major Sections)**: 22-24pt, Bold, BNZ Navy Blue
- **H3 (Subsections)**: 18-20pt, Bold, Dark Gray
- **H4 (Minor Headings)**: 16pt, Bold, Dark Gray
- **Body Text**: 11-12pt, Regular, Dark Gray
- **Captions**: 9-10pt, Regular, Medium Gray

#### Presentations (PowerPoint)
- **Slide Title**: 32-36pt, Bold, BNZ Navy Blue
- **Section Header**: 28pt, Bold, BNZ Orange
- **Body Text**: 18-20pt, Regular, Dark Gray
- **Bullet Points**: 16-18pt, Regular, Dark Gray
- **Captions/Footnotes**: 12-14pt, Regular, Medium Gray

#### Diagrams (Draw.io)
- **Diagram Title**: 24-28pt, Bold, BNZ Navy Blue
- **Section Headers**: 18-20pt, Bold, Dark Gray
- **Box Labels**: 12-14pt, Bold, Dark Gray
- **Annotations**: 10-11pt, Regular, Medium Gray
- **Legend Text**: 10-11pt, Regular, Dark Gray

### Typography Rules

#### ✅ DO:
- Use **Calibri** for all documents and presentations
- Use **Helvetica** for diagrams
- Use **bold** for headings and emphasis
- Use **italic** sparingly for quotes or subtle emphasis
- Maintain **1.5 line spacing** for body text
- Use **sentence case** for most headings (not ALL CAPS)

#### ❌ DON'T:
- Don't use more than 2 font families in a single artifact
- Don't use font sizes smaller than 9pt (readability)
- Don't use ALL CAPS for long text (accessibility issue)
- Don't use decorative fonts (Comic Sans, Papyrus, etc.)
- Don't use colored text as the only indicator (accessibility)

## Diagram Standards

### Draw.io Diagram Requirements

#### Canvas Settings
- **Page Size**: 2400x1800px (16:9 ratio) or 1920x1080px (standard HD)
- **Grid**: 10px grid, snap enabled
- **Background**: White (#FFFFFF) or Background Gray (#F5F5F5)
- **Margin**: 40-50px on all sides

#### Visual Hierarchy

**Level 1: Primary Elements** (Most Important)
- **Fill Color**: BNZ Navy Blue (#003087)
- **Font**: 18-20pt Bold, White text
- **Stroke**: 2px, Dark Gray
- **Examples**: Main system boxes, key decisions

**Level 2: Secondary Elements**
- **Fill Color**: BNZ Light Blue (#50E6FF) or Light Gray (#CCCCCC)
- **Font**: 14-16pt Bold, Dark Gray text
- **Stroke**: 2px, Medium Gray
- **Examples**: Supporting systems, processes

**Level 3: Tertiary Elements**
- **Fill Color**: Background Gray (#F5F5F5) or White
- **Font**: 12-14pt Regular, Dark Gray text
- **Stroke**: 1px, Light Gray
- **Examples**: Details, annotations, notes

#### Shape Standards

| Shape Type | Use Case | Styling |
|------------|----------|---------|
| **Rectangle** | Systems, components, entities | Rounded corners (arcSize=10) |
| **Diamond** | Decision points | Sharp corners, centered text |
| **Circle/Ellipse** | Start/end points, actors | Centered text, bold border |
| **Cylinder** | Databases, data stores | Standard database symbol |
| **Hexagon** | External systems, APIs | Angled sides, centered text |
| **Parallelogram** | Data/documents | Slanted sides, left-aligned text |
| **Cloud** | Cloud services | Rounded cloud shape |
| **Arrow** | Data flow, relationships | 2px width, solid or dashed |

#### Arrow/Connector Standards

| Arrow Type | Line Style | Use Case |
|------------|------------|----------|
| **Solid Arrow** | 2px, solid | Primary data flow, required relationships |
| **Dashed Arrow** | 2px, dashed (5-5) | Optional flow, feedback loops |
| **Double Arrow** | 2px, solid, both ends | Bidirectional communication |
| **No Arrow** | 2px, solid, no ends | Grouping, container boundaries |

**Arrow Colors**:
- **BNZ Navy Blue**: Primary flows
- **BNZ Orange**: Critical/priority flows
- **BNZ Teal**: Success/positive flows
- **Dark Gray**: Standard relationships

#### Legend Requirements

Every diagram MUST include a legend with:
- **Color meanings** (if using color coding)
- **Shape meanings** (if using multiple shape types)
- **Arrow types** (if using different arrow styles)
- **Status indicators** (if showing status)

Legend should be:
- Positioned in bottom-left or bottom-right corner
- Background: Light Gray (#F5F5F5) with 1px border
- Font: 10-11pt Regular
- Size: Compact but readable

#### Diagram Types & Templates

**1. Architecture Diagrams**
- Use rectangles for systems
- Use cylinders for databases
- Use hexagons for external APIs
- Use clouds for cloud services
- Group related components with containers
- Show data flow with arrows
- Include legend and key

**2. Process Flow Diagrams**
- Use rectangles for process steps
- Use diamonds for decisions
- Use circles for start/end
- Use arrows for flow direction
- Include swimlanes if multiple actors
- Label decision branches (Yes/No, Success/Fail)

**3. Data Flow Diagrams**
- Use parallelograms for data
- Use rectangles for processes
- Use cylinders for data stores
- Use arrows with labels
- Show data transformation clearly
- Include data volume/frequency if relevant

**4. Organizational/Hierarchy Diagrams**
- Use rectangles for roles/teams
- Use tree structure for reporting lines
- Use consistent sizing for same level
- Color-code by department/function
- Include names/titles

**5. Comparison Matrices**
- Use table structure
- Header row: BNZ Navy Blue background, white text
- Alternating row colors: White and Background Gray
- Cell borders: 1px Light Gray
- Use semantic colors for ratings (Green=Good, Yellow=Medium, Red=Poor)

#### Common Diagram Patterns

**Pattern 1: Layered Architecture**
```
┌─────────────────────────────────────┐
│  Experience Layer (Light Blue)      │
├─────────────────────────────────────┤
│  Engagement Layer (Navy Blue)       │
├─────────────────────────────────────┤
│  Data Access Layer (Teal)           │
├─────────────────────────────────────┤
│  Infrastructure Layer (Gray)        │
└─────────────────────────────────────┘
```

**Pattern 2: Hub-and-Spoke**
- Central hub: Navy Blue, larger box
- Spokes: Light Blue, smaller boxes
- Connections: Solid arrows from hub to spokes

**Pattern 3: Pipeline/Sequential**
- Left-to-right flow
- Equal-sized boxes
- Numbered steps (1, 2, 3...)
- Arrows between each step

#### Accessibility Requirements

- **Contrast**: Minimum 4.5:1 for text on backgrounds
- **Color Independence**: Don't rely on color alone (use patterns/labels)
- **Text Size**: Minimum 10pt for all text
- **Alt Text**: Provide description in diagram metadata
- **Shape Labels**: Always label shapes, not just color-code
